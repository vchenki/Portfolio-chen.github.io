物栈

一个帮助用户系统化管理长期物品与资产的信息工具，围绕「物品 → 分类 → 位置 → 提醒」的核心关系建模。

信息架构 Next.js 数据可视化 本地存储

Problem & Solution

Problem

随着物品增多，"东西放哪了""这个还需要吗""什么时候买的"成为日常痛点。电子表格无法表达位置关系和分类层级，通用笔记工具又缺乏物品管理所需的专业字段。

Solution

围绕「物品 → 分类 → 位置 → 提醒」的核心关系建模，提供结构化的物品录入、多维检索与可视化统计，让用户真正掌控自己的物品资产。

Key Interfaces

首页 / 总览界面
物品分类统计、位置分布与提醒事项一目了然
分类管理 / 物品组织
多级分类体系与标签组织，支持更清晰的物品归类与检索
数据分析 / 统计视图
通过图表查看数量分布、价值估算与使用频率，辅助长期管理决策

Core Features

  • ◆
    多级分类体系 — 支持父类 / 子类 / 标签三层结构，适应不同整理维度
  • ◆
    空间位置标记 — 以「区域 → 具体位置」的层级记录物品存放位置
  • ◆
    主动提醒系统 — 基于时间或使用频率提醒维护，防止遗忘
  • ◆
    统计分析面板 — 数量、分类分布、价值估算等数据一目了然

Technical Decisions

  • ◆
    本地 SQLite — 所有数据本地存储，无隐私风险，支持离线使用
  • ◆
    ECharts 可视化 — 复用成熟图表库，快速实现统计面板
  • ◆
    CSV 导入导出 — 降低数据迁移成本，用户不被工具绑定

Reflection

What Worked
  • 信息架构设计满足了一年内迭代的功能扩展需求
  • 位置 + 分类双维检索在实际使用中很好用
  • 提醒系统的触发逻辑足够灵活
Next Steps
  • 移动端适配体验较差，响应式需要重做
  • 缺乏批量操作，录入效率低
  • 图片管理能力弱，物品照片存储方案需优化
